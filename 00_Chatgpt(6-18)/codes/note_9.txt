# ‚úÖ **3. ENCAPSULATION**

Hiding data using **private** variables + getters/setters.

### ‚úî Code


# ‚úÖ **5. POLYMORPHISM**

### ‚úî Compile-time Polymorphism = Method Overloading

(Already explained in string code but here is OOP example)

```java
class Adder {

    int add(int a, int b) { return a + b; }
    int add(int a, int b, int c) { return a + b + c; }
}

public class OverloadDemo {
    public static void main(String[] args) {
        Adder obj = new Adder();
        System.out.println(obj.add(5, 10));
        System.out.println(obj.add(5, 10, 20));
    }
}
```

### OUTPUT

```
15
35
```

---

### ‚úî Runtime Polymorphism = Method Overriding

```java
class Vehicle {
    void run() { System.out.println("Vehicle running"); }
}

class Bike extends Vehicle {
    @Override
    void run() { System.out.println("Bike running"); }
}

public class OverrideDemo {
    public static void main(String[] args) {
        Vehicle v = new Bike();   // parent reference, child object
        v.run();                  // runtime method dispatch
    }
}
```

### OUTPUT

```
Bike running
```

# ‚úÖ **1. ACCESS MODIFIERS in Java**

| Modifier      | Access Within Class | Same Package | Subclass | Other Package |
| ------------- | ------------------- | ------------ | -------- | ------------- |
| **private**   | ‚úî                   | ‚ùå            | ‚ùå        | ‚ùå             |
| **default**   | ‚úî                   | ‚úî            | ‚ùå        | ‚ùå             |
| **protected** | ‚úî                   | ‚úî            | ‚úî        | ‚ùå             |
| **public**    | ‚úî                   | ‚úî            | ‚úî        | ‚úî             |

---

## ‚úî Example showing all modifiers

```java
class TestModifiers {

    private int a = 10;       // only within class
    int b = 20;               // default ‚Üí same package
    protected int c = 30;     // package + inherited
    public int d = 40;        // everywhere

    public void show() {
        System.out.println(a + " " + b + " " + c + " " + d);
    }
}

public class AccessModifiersDemo {
    public static void main(String[] args) {
        TestModifiers t = new TestModifiers();
        t.show();
    }
}
```

### OUTPUT

```
10 20 30 40
```

# ‚úÖ **3. DESTRUCTOR in Java? (Important Concept)**

üí° **Java does NOT support destructors**
Because Java has **Garbage Collector (GC)** which automatically destroys objects.

But Java has a special method:

### ‚úî finalize() ‚Äî Called before destruction (NOT recommended now)

```java
public class DestructorDemo {
    @Override
    protected void finalize() {
        System.out.println("Object destroyed by GC");
    }

    public static void main(String[] args) {
        DestructorDemo d = new DestructorDemo();
        d = null;
        System.gc();   // request GC
    }
}
```

### OUTPUT (may vary)

```
Object destroyed by GC
```

# ‚úÖ **7. super keyword**

### ‚úî Access parent variables & methods

```java
class Parent {
    int x = 10;

    void message() {
        System.out.println("Parent message");
    }
}

class Child extends Parent {
    int x = 20;

    void display() {
        System.out.println(super.x);  // parent variable
        System.out.println(this.x);   // child variable
        super.message();              // parent method
    }
}

public class SuperKeywordDemo {
    public static void main(String[] args) {
        new Child().display();
    }
}
```

### OUTPUT

```
10
20
Parent message
```
