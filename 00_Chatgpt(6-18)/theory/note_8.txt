## **1. Definition**

A String in Java is a **sequence of characters** represented as an object of the `String` class.
Unlike many languages where strings are primitive, Java treats Strings as **immutable objects** stored in the heap.

---

## **2. Memory Behavior (Deep Theory)**

### **2.1 Immutable Nature**

Once a String object is created, **its value cannot be changed**.
Any modification produces a **new String object**.

### **2.2 String Constant Pool (SCP)**

Java maintains a special memory region called **String Pool**, where literal strings are stored.

Rules:

* Duplicate literals are not created; reference is reused.
* Immutable nature enables pooling, saving memory and improving performance.

### **2.3 Heap Memory**

Strings created using constructors or new operations are stored in the **heap**, not the pool.

### **2.4 Why Strings Are Immutable**

* Security (class names, passwords, URLs)
* Thread-safety
* HashCode caching
* Memory optimization through pooling

---

## **3. Important Properties of Strings**

### **3.1 Immutable**

Cannot be altered once created.

### **3.2 Stored as Objects**

String is a **reference type**, not primitive.

### **3.3 Interning**

SCP uses interning to avoid duplicates.

### **3.4 Comparable**

Strings implement **Comparable**, enabling lexicographical comparison.

### **3.5 Length Fixed After Creation**

Once created, the sequence length does not change.

### **3.6 Unicode-Based**

Java Strings internally use Unicode, supporting all languages and symbols.

### **3.7 Backed by a Character Array**

String value is internally stored using an array-like structure (conceptually char[]).
